# Source Code (`src/`)

This directory contains all the production-ready Python scripts responsible for the Data Science and Machine Learning pipelines of the **predict-traffic-volume** project. 

Notebooks in this repository are exclusively used for exploration and prototyping. The actual execution, data processing, model training, and metric extraction must be done using the scripts below.

---

## Directory Structure & Modules

### 1. `data/`
Responsible for the initial ingestion and structural cleaning of the raw dataset.
- **`preprocessing.py`**: The core data cleaner. It handles datetime parsing, extraction of boolean features (e.g., `is_raining`, `is_snowing`, `is_rush_hour`), applies cyclical encoding (sine and cosine transformations for hours and days) to capture temporal patterns, and interpolates abnormal numerical anomalies (like zero Kelvin temperatures).

### 2. `run_pipeline.py`
- **The Data Preprocessing Trigger**: This script orchestrates the pipeline from raw to clean data. It reads the raw `.csv.gz`, instantiates the processor from `preprocessing.py`, and outputs a consolidated `clean_traffic_data.csv` to `data/processed/`.

### 3. `models/`
The heart of the Machine Learning architecture. Everything related to data splitting, algorithms, and evaluation lives here.
- **`features_config.py`**: A centralized configuration file specifying which features to drop (to avoid multicollinearity or noise) and which numerical features require scaling.
- **`data_splitter.py`**: A strict module to handle chronological splitting (Train, Validation, and Test sets) preventing Look-Ahead Bias. It also applies `SimpleImputer` and `StandardScaler` fitted *strictly* on the Training partition to prevent Data Leakage.
- **`algorithms.py`**: The "Model Factory". It centralizes the instantiation of all supported regression algorithms (Ridge, Random Forest, XGBoost, LightGBM, CatBoost, and MLP) configuring basic parameters (e.g., seeds and parallelization limits).
- **`evaluator.py`**: The academic math engine. Computes robust regression metrics (RMSE, MAE, R², MAPE, Max Error) and saves semantic JSON logs with timestamps inside `reports/logs/`.
- **`train_model.py`**: The **Master Orchestrator**. It loads the partitioned data, triggers the Bayesian Optimization (`Optuna`) paired with `TimeSeriesSplit`, trains the final tuned models, evaluates them, and serializes the best estimators (`.pkl`) to the `models/` directory.
- **`plot_metrics.py`**: The visualizer. Scans the JSON logs generated by `train_model.py` and creates publication-ready plots (Bar charts for metrics comparison and Time Series line plots for Actual vs. Predicted Traffic) inside `reports/figures/`.

---

## How to Run the Pipeline

All scripts should be executed from the root directory of the project using `poetry` to ensure dependency isolation.

### Step 1: Preprocess the Raw Data
Run this first to generate the `clean_traffic_data.csv`.
```bash
poetry run python src/run_pipeline.py
```

### Step 2: Split and Scale the Data
Partitions the data chronologically and applies isolated scaling.
```bash
poetry run python src/models/data_splitter.py
```

### Step 3: Train and Tune the Models (The Orchestrator)
You can train a single model, a few models, or all of them. The orchestrator uses Optuna for Hyperparameter Tuning.
- **Train a single model with 10 Optuna trials:**
  ```bash
  poetry run python src/models/train_model.py --mode validation --models xgboost --trials 10
  ```
- **Train specific models:**
  ```bash
  poetry run python src/models/train_model.py --mode validation --models xgboost random_forest lightgbm
  ```
- **Run the Final Test on the Golden Vault (Test Set):**
  *(Run this ONLY once you have decided on the absolute best model).*
  ```bash
  poetry run python src/models/train_model.py --mode test --models random_forest --trials 50
  ```

### Step 4: Generate Visual Reports
After generating logs, plot the metrics and the actual vs. predicted time series.
```bash
poetry run python src/models/plot_metrics.py
```
*The resulting graphs will be saved in `reports/figures/`.*
