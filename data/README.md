# Data Directory

This directory stores all datasets used and generated throughout the `predict-traffic-volume` project. It is structured to maintain a clear separation between immutable raw data and derived files, ensuring reproducibility and preventing data corruption.

## Directory Structure & Files

### `raw/`
Contains the original, immutable data as downloaded from the source (UCI Machine Learning Repository). **These files should never be modified by code.**
*   `Metro_Interstate_Traffic_Volume.csv`: The primary raw dataset containing hourly traffic volume, weather, and holiday data from 2012 to 2018.

### `processed/`
Contains the final, cleaned data that has passed through the preprocessing and feature engineering pipelines. These files are ready for modeling.
*   `clean_traffic_data.csv`: The intermediate dataset after missing values imputation, outlier removal, gap dropping, and feature engineering (cyclical time coordinates, binary weather flags). 

**Machine Learning Split Sets (Generated by `src/models/data_splitter.py`):**
To ensure perfect reproducibility and to prevent Data Leakage during modeling, `clean_traffic_data.csv` is chronologically split and scaled, generating the following explicitly named files:

*   **Training Set (70%):**
    *   `X_train_scaled.csv`: The independent variables (features) used to train the models. Numerical columns (like temperature) have been fitted and transformed using a Standard Scaler.
    *   `y_train.csv`: The target variable (`traffic_volume`) corresponding to the training features.

*   **Validation Set (15%):**
    *   `X_val_scaled.csv`: The features used to tune hyperparameters and select the best model. Scaled strictly using the scaler fitted on the Training Set.
    *   `y_val.csv`: The target variable for the validation features.

*   **Test Set (15%):**
    *   `X_test_scaled.csv`: The completely unseen features used exclusively for the final, unbiased evaluation of the chosen model. Scaled strictly using the scaler fitted on the Training Set.
    *   `y_test.csv`: The target variable for the testing features.
